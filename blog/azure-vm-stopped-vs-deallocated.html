Azure VM Stopped vs Deallocated:
The Billing Difference That Costs Thousands

Your VM is off. The OS is shut down. So why is Azure still charging you? Because "stopped" and "deallocated" are not the same thing — and the difference can silently drain your budget every single month.

S
Suresh · CleanCloud
· March 2026 · 6 min read · Azure · FinOps

The Trap Every Azure Engineer Falls Into

You're tidying up a dev environment. You SSH into a VM and run sudo shutdown now. Job done — the machine is off, right? Costs stopped?

Wrong. Azure will keep billing you for that VM's compute allocation until the end of time, or until someone notices.

This is one of the most common and most surprising Azure billing traps. It's not a bug. It's intentional design — but it's design that costs teams real money when they don't know about it.

⚠️ A VM shut down from inside the OS enters "Stopped" state — not "Stopped (deallocated)". Azure continues to charge full compute rates. Only deallocated VMs stop compute billing.

What "Stopped" Actually Means

When an Azure VM enters the Stopped state, the guest operating system has shut down. The VM is not running. But from Azure's perspective, the underlying compute resources — the CPU cores, memory, and network interface reservation — are still allocated to your subscription.

Azure holds that capacity reserved for you. And you pay for that reservation, whether you use it or not.

A common question when researching Azure VM stopped vs deallocated cost: does a stopped Microsoft Azure Virtual Machine still charge you? The answer is yes — and the billing difference between the two states is the most misunderstood aspect of Azure compute pricing.

State Compute Charges Storage (Disk) Public IP How you get here
Running ✅ Full rate ✅ Yes ✅ Retained Normal operation
Stopped (still billing!) ✅ Full rate ✅ Yes ✅ Retained OS shutdown, shutdown /s, sudo shutdown
Stopped (deallocated) (safe) ❌ No compute charge ✅ Yes ⚠️ Dynamic IPs lost Portal Stop, az vm deallocate, PowerShell default

⚠️ Azure VM stopped still charging? If your Microsoft Azure Virtual Machine shows "Stopped" (not "Stopped (deallocated)") in the portal, you are being billed full compute rates. This is the az_stopped_vm finding CleanCloud surfaces with HIGH confidence.

How Engineers Accidentally Stop (Not Deallocate)

There are three main ways teams end up with VMs in the costly Stopped state:

  • OS-level shutdown: sudo shutdown -h now on Linux, or shutdown /s on Windows. The OS powers off, but Azure doesn't know you intended to release resources.
  • PowerShell with wrong flag: Stop-AzVM -Name myVM -ResourceGroupName myRG -StayProvisioned — the -StayProvisioned flag explicitly keeps the compute allocated.
  • Automation scripts from before 2018: Older Azure automation runbooks used Stop-AzureVM (the classic cmdlet) which defaulted to staying provisioned.

"The machine is off. Why is the bill still on?" — Every Azure engineer, at least once.

How to Properly Deallocate a VM

The good news: it's easy once you know. There are three correct ways to stop a VM and stop billing.

Azure CLI (safest for scripts)

az vm deallocatebash
# Deallocate a single VM
az vm deallocate \
  --resource-group my-resource-group \
  --name my-vm-name

# Deallocate all VMs in a resource group
az vm list \
  --resource-group my-resource-group \
  --query "[].name" -o tsv | \
  xargs -I{} az vm deallocate \
    --resource-group my-resource-group \
    --name {}

# Verify state after — should show "deallocated"
az vm show \
  --resource-group my-resource-group \
  --name my-vm-name \
  --show-details \
  --query "powerState"

PowerShell

Stop-AzVMPowerShell
# Default behaviour: deallocates (no -StayProvisioned)
Stop-AzVM `
  -ResourceGroupName "my-resource-group" `
  -Name "my-vm-name"

# DANGER: this stops but keeps compute allocated (still billed!)
# Stop-AzVM -ResourceGroupName "myRG" -Name "myVM" -StayProvisioned

Azure Portal

Clicking Stop in the Azure Portal deallocates the VM by default since 2018. The VM status will show as Stopped (deallocated) in the portal. If it just shows Stopped without the "(deallocated)" suffix — you're still being billed.
